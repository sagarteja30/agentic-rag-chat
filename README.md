A production-ready **Retrieval-Augmented Generation (RAG)** system with intelligent agentic planning for document analysis. Built with FastAPI, Streamlit, Groq LLM, and Supabase.

## 🌟 Features

- **Intelligent Agentic Planning** - Multi-step reasoning with execution plans
- **Production-Ready API** - Async processing, rate limiting, caching, health checks
- **Multi-Format Support** - PDF, DOCX, CSV, TXT, Markdown, and more
- **Advanced Text Chunking** - Semantic chunking with metadata
- **ChatGPT-Style UI** - Beautiful dark-mode Streamlit interface
- **Conversation Management** - Multiple chat sessions with history
- **Vector Search** - Semantic similarity search with Supabase
- **Confidence Scoring** - Quality indicators for responses

## 🚀 Quick Start

### Prerequisites

- Python 3.11+
- Supabase account (free tier)
- Groq API key (free tier)

### Installation

```bash
# Clone the repository
git clone <your-repo-url>
cd Agentic_RAG_BOT

# Create virtual environment
python -m venv venv

# Activate on Windows
venv\Scripts\activate

# Activate on Mac/Linux
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

### Configuration

Create a `.env` file in the project root:

```env
# Supabase Configuration
SUPABASE_URL=your_supabase_project_url
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key

# Groq API Key (Get from https://console.groq.com)
GROQ_API_KEY=your_groq_api_key

# API Settings
API_BASE_URL=http://localhost:8000

# Embedding Model (default)
EMBEDDING_MODEL=sentence-transformers/all-MiniLM-L6-v2

# Optional Settings
TOP_K=8
MAX_CONTEXT_LENGTH=6000
```

### Setup Supabase Database

Run this SQL in your Supabase SQL Editor:

```sql
-- Enable vector extension
create extension if not exists vector;

-- Create documents table with vector column
create table if not exists documents (
    id bigint generated by default as identity primary key,
    doc_id text not null,
    chunk_id integer not null,
    content text not null,
    metadata jsonb default '{}',
    embedding vector(384),
    created_at timestamptz default now()
);

-- Create index for vector similarity search
create index on documents using ivfflat (embedding vector_cosine_ops) with (lists = 100);

-- Enable Row Level Security
alter table documents enable row level security;

-- Create RPC function for vector matching
create or replace function match_documents(
    query_embedding vector(384),
    match_count int default 8,
    match_threshold float default 0.05
)
returns table (
    id bigint,
    doc_id text,
    chunk_id integer,
    content text,
    metadata jsonb,
    similarity_score float
)
language plpgsql
as $$
begin
    return query
    select
        d.id,
        d.doc_id,
        d.chunk_id,
        d.content,
        d.metadata,
        1 - (d.embedding <=> query_embedding) as similarity_score
    from documents d
    where 1 - (d.embedding <=> query_embedding) > match_threshold
    order by d.embedding <=> query_embedding
    limit match_count;
end;
$$;
```

### Ingest Documents

```bash
# Put your documents in the `documents` folder
# Supported: PDF, DOCX, TXT, CSV, MD, etc.

python ingest_in_db.py --clear
```

### Run the Application

```bash
# Terminal 1: Start FastAPI Backend
python agentic_rag.py

# Terminal 2: Start Streamlit Frontend
streamlit run agentic_rag_streamlit.py
```

Open http://localhost:8501 in your browser.
